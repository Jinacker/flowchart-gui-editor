## 2. Graphviz 레이아웃 도입

기존 BFS 수동 레이아웃을 **Graphviz dot 엔진**으로 교체.

### 배경

BFS 레이아웃의 한계:
- 방향 변경 시 좌표 재계산 없음
- 교차 엣지 최소화 미지원
- 복잡한 사이클 그래프에서 배치 불안정

Graphviz는 rank 기반 계층 배치 + 교차 최소화 + 사이클 처리를 내부적으로 해결.

### 구현 내용

#### 패키지 추가

```
@hpcc-js/wasm-graphviz v1.21.2
```

WASM이 `dist/index.js`에 인라인으로 번들됨 → 별도 WASM 파일 배포 불필요.
`vite.config.js` 수정 없이 기존 IIFE 빌드에 그대로 통합.

#### 신규 파일: `src/composables/graphvizLayout.js`

역할: `assignLayout()` 을 대체하는 Graphviz 기반 좌표 계산기.

주요 구현:
- `Graphviz.load()` 싱글턴 (WASM 로드 최초 1회)
- `toDot()`: rawNodes/rawEdges → DOT 문자열 변환
  - Mermaid direction → Graphviz `rankdir` 매핑 (TD→TB, LR→LR, BT→BT, RL→RL)
  - Mermaid shape → Graphviz shape 매핑
  - `&quot;` 등 HTML 엔티티 디코딩 처리 (LLMService 이슈 부분 해결)
  - 노드 크기 `fixedsize=true` (NODE_W_INCH=1.8, NODE_H_INCH=0.55) — BFS 기존 크기와 맞춤
  - `nodesep=1.0, ranksep=1.3` — BFS 기존 간격(CROSS_GAP=100, DEPTH_GAP=130)과 맞춤
- `parsePlain()`: Graphviz plain 포맷 출력 파싱 (quoted 토큰 처리 포함)
- 좌표 변환: Graphviz(중심 기준, y 하단 원점) → VueFlow(좌상단 기준, y 상단 원점)

#### 수정 파일: `src/composables/useMermaidParser.js`

- `layoutWithGraphviz` import 추가
- `assignLayoutBfs` 함수 전체 제거 (Graphviz로 대체, 폴백 불필요)
- `parseMermaid` → `async function` 으로 변경
- `assignLayout()` 호출 → `await layoutWithGraphviz()` 로 교체

#### 수정 파일: `src/MermaidFlowEditor.vue`

- `onMounted(() => {})` → `onMounted(async () => {})` 변경
- `parseMermaid()` → `await parseMermaid()` 변경

### 빌드 결과

```

mermaid-flow-editor.iife.js  1,043.46 kB │ gzip: 708.64 kB
```

번들 크기 약 350KB 증가 (WASM 인라인). 별도 파일 배포 없이 자급자족 유지.

### BFS 폴백 제거 이유

WASM이 번들에 인라인 포함되어 있어 JS 로드 성공 시 Graphviz는 항상 사용 가능.
BFS 폴백은 실제로 도달할 수 없는 코드(dead code)였으므로 제거.
에러 발생 시에는 `MermaidFlowEditor.vue`의 `catch`에서 빈 캔버스로 폴백.
